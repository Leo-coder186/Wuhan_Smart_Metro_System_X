# 武汉智慧地铁三维可视化管控平台 — 答辩稿

> 答辩时长建议：8-10 分钟讲解 + 5 分钟演示 + 5 分钟提问

## 一、开场（30秒）

各位老师好，我的毕业设计题目是**武汉智慧地铁三维可视化管控平台**。

这是一个基于 CesiumJS 三维地球引擎和 Vue 3 框架构建的地铁运营可视化系统，将武汉地铁的线路、站点、列车运行状态以三维方式呈现在大屏上，支持实时监控和数据分析。

---

## 二、项目背景（1分钟）

### 为什么做这个？

- **武汉地铁现状**：运营里程超过 500 公里，日均客流量超 400 万人次，传统二维监控系统难以直观展示全网运行状态
- **行业趋势**：数字孪生、智慧城市是当前热点，三维可视化是地铁运营管理的升级方向
- **技术可行性**：CesiumJS 开源三维引擎成熟，WebGL 硬件加速普及，前端框架完善

### 解决了什么问题？

1. 把平面的地铁线路图变成**可交互的三维场景**
2. 把分散的数据（客流量、告警、列车位置）**聚合到一个大屏**
3. 把抽象的数据变成**可视化的图表和热力图**

## 四、核心功能演示（4分钟）★ 最重要

### 演示顺序（建议按这个来）

#### 1. 首页 — 全局概览（1分钟）

**展示内容：**
- 武汉三维地图 + 建筑白模 + 天空盒
- 6 个数据面板：运营统计、客流指标、告警趋势、地铁活动、线路概览、实时影像
- 中间透明区域可拖拽旋转缩放地图

**讲解要点：**
> "首页采用 CSS Grid 布局，中间有意留空让底图透出。UI 层设置 pointer-events: none，只有面板区域可交互，其余点击穿透到底图。"

#### 2. 地铁线路渲染（30秒）

**展示：** 点击底栏回到首页，地图上的彩色泛光地铁线路

**讲解要点：**
> "线路使用 Cesium 的 PolylineGlowMaterialProperty 实现泛光效果。数据从后端 API 获取，经过 GCJ02→WGS84 坐标转换后渲染。"

#### 3. 站点管理 — 热力图（1分钟）★ 亮点

**操作：** 底栏 → 站点管理 → 左侧选"站点拥挤度"

**展示：** 地图上出现红蓝渐变的圆形热力标记

**讲解要点：**
> "热力图的实现用了三层：heatmap.js-fixed 在离屏 Canvas 上绘制 → 将 Canvas 转为 Cesium 材质 → 通过 Rectangle 几何体叠加到地球表面。相机拉近时热力点半径自动增大。"

#### 4. 实时轨迹（1分钟）

**操作：** 底栏 → 实时轨迹 → 点击线路

**展示：** 地图上出现列车模型 + 气泡弹窗，时间轴播放列车运行

**讲解要点：**
> "列车模型是地铁 glTF 模型。位置数据目前用模拟数据，真实项目中通过 WebSocket 从后端实时推送，使用 Cesium 的 SampledProperty 实现平滑移动。"

#### 5. 发展历程（30秒）

**操作：** 底栏 → 发展历程 → 点击线路 → 播放

**展示：** 地图只显示该线路，动画沿路径漫游

## 五、技术难点与解决方案（2分钟）★ 加分项

### 难点 1：Cesium 在大屏缩放容器中的交互问题

**问题：** v-scale-screen 使用 CSS transform:scale() 缩放整个页面，Cesium 的 canvas 在缩放过程中尺寸短暂归零，导致渲染崩溃 `"Expected width to be greater than 0"`

**解决：** 对 CesiumWidget 的 render 方法做了安全包装（monkey-patch），在每帧渲染前检查 canvas 尺寸，为 0 时跳过本帧

```js
cesiumWidget.render = function () {
    const canvas = viewer.canvas;
    if (!canvas || canvas.clientWidth === 0 || canvas.clientHeight === 0) return;
    try { return _originalRender(); }
    catch (e) { /* 吞掉非致命渲染错误 */ }
};
```

### 难点 2：UI 层与 3D 图层的点击事件分层

**问题：** UI 面板和 Cesium 底图叠加在一起，需要精确控制哪些区域响应点击

**解决：** 设计了 Z-Index + pointer-events 分层方案

| 层级 | z-index | pointer-events | 作用 |
|------|---------|:---:|------|
| Footer | 200 | all | 底栏导航 |
| router-wrapper | 100 | none | 子页面容器(透传) |
| 面板内容 | 继承 | all | 图表/按钮可交互 |
| #home | 10 | none | UI 根层(透传) |
| Cesium | 1 | all | 底图可交互 |

### 难点 3：GCJ02 与 WGS84 坐标转换

**问题：** 后端存储的是国测局 GCJ02 坐标，Cesium 使用 WGS84，直接渲染会偏移 300-500 米

**解决：** 在数据加载阶段统一调用 CoordTransform 转换

## 八、可能的提问及回答

**Q: 为什么选择 Cesium 而不是 Three.js？**
> Cesium 专为地理信息系统设计，内置 WGS84 坐标系统、3D Tiles 海量模型加载、影像图层切换。Three.js 是通用 3D 引擎，做 GIS 需要自己实现坐标投影、地形等，工作量巨大。

**Q: 热力图的性能如何？**
> 热力图是一张静态 Canvas 贴图叠加在地球表面，不随数据量增加而降低帧率。数据量只影响 Canvas 绘制耗时（毫秒级）。

**Q: 3D 建筑模型的数据来源？**
> 武汉城市建筑白模，通过 Cesium 3D Tiles 格式加载，使用 CustomShader 实现高度渐变着色和昼夜切换。

**Q: 如何处理大屏分辨率的适配？**
> v-scale-screen 组件将 1920×1080 的设计稿等比缩放，所有尺寸使用 vw 单位，确保在任何分辨率下比例一致。
